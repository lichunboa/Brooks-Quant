# 项目路线与操作规范

## 1. 总原则

- 研究入口统一使用 `CTA回测`
- 运行入口统一使用 `CTA策略`
- 图表核对入口统一使用 `Brooks图表`
- 多品种批量脚本只作为补充研究工具，不作为最终结论来源

## 2. 理论引用顺序

后续所有策略逻辑、知识点定义、参数说明、优化方向，统一按以下顺序引用：

1. `策略资料/al brooks参考资料agent专用版 /课程`
2. `策略资料/al brooks参考资料agent专用版 /百科`
3. `策略资料/al brooks参考资料agent专用版 /Ali参考`

说明：

- `太妃价格行为` 不再作为主理论依据
- 课程里没有明确写法时，才用百科或 Ali 补细节

## 3. 三个模块怎么配合

### CTA回测

用途：

- 研究单策略、单品种、单执行周期
- 看统计、委托、成交、每日盈亏、K 线图
- 做参数优化

补充：

- 现已支持 `本地数据集` 下拉框
- 选择数据集后会自动带入：
  - `本地代码`
  - 默认 `1m`
  - 推荐的 `手续费率 / 滑点 / 价格跳动 / 回测资金`

### CTA策略

用途：

- 把已经稳定的策略实例化
- 初始化、启动、停止
- 做模拟盘或实盘运行

补充：

- 现已支持 `本地数据集` 下拉框
- 添加策略时会自动把所选数据集的 `vt_symbol` 预填进弹窗

### Brooks图表

用途：

- 审计信号是否算对
- 对照回测记录看入场、止损、目标和生命周期
- 以后逐条核对知识点

当前补充：

- 图表默认显示 `EMA20`
  - 口径按 Al Brooks 默认 `MA = 20 bar EMA`
  - 当前使用收盘价计算
- 图表支持直接调整 K 线图的宽和高
  - `宽压 / 宽松`：调可见 K 线数量
  - `高压 / 高松`：调纵向价格压缩程度
  - 鼠标放到右侧价格轴滚轮：直接调图高
  - 鼠标放到底部时间轴滚轮：直接调图宽
- 下方时间轴已改成更短的时间标签
  - 目的是在同样宽度下显示更多时间刻度
- `知识点 -> 背景 -> 突破`
  - 现已改成双层模型：
  - 底层结构：`震荡 / 窄幅通道 / 宽幅通道`
  - 上层事件：`突破起爆 / 突破跟进 / 突破测试 / 失败突破`
  - 这样不会把通道里的每次推进都当突破，也不会让实盘只能看到很晚的确认
  - `背景总览` 不再整屏铺竖条，而是：
  - 结构层用 `TR / Tight CH / Broad CH` 局部标签
  - 事件层用 `BO / FT / Test / FBO` 局部框

## 4. 我们以后怎么做

固定顺序：

1. 在 `CTA回测` 里选策略、品种、执行周期
2. 跑回测
3. 回测结果自动同步到 `Brooks图表 -> 记录`
4. 在 `Brooks图表 -> 知识点` 里按知识点逐项核对
5. 修正信号逻辑或参数
6. 回到 `CTA回测` 重跑
7. 稳定后再放进 `CTA策略`

## 5. CTA回测怎么跑

### 第一次使用

1. 打开 `CTA回测`
2. 点击 `策略重载`
3. 在 `交易策略` 下拉框里找：
   - `Ema20H2L2TrendStrategy`
   - `H1L1FirstPullbackStrategy`
   - `Mag20GapStrategy`

如果下拉框里没看到，先确认当前是从仓库根目录启动应用，再点一次 `策略重载`。

### 单次回测步骤

1. 选择 `交易策略`
2. 优先从 `本地数据集` 下拉框选择品种
3. 如有需要，再手工改 `本地代码`
   - 例如：`US500.OTC`、`NAS100.OTC`、`BTCUSDT.GLOBAL`
4. `K线周期` 统一先填 `1m`
5. `开始日期`、`结束日期` 填研究窗口
6. 检查自动带入的回测参数：
   - 加密：
     - `手续费率 = 0.0005`
     - `交易滑点 = 0.01` 或按实际品种最小跳动
     - `合约乘数 = 1`
     - `价格跳动 = 0.01 / 0.001 / 0.0001 / 0.00001` 按品种
     - `回测资金 = 100000`
   - 外汇/指数/金银：
     - `手续费率 = 0`
     - `交易滑点 = 价格跳动`
     - `合约乘数 = 1`
     - `回测资金 = 100000`
7. 点 `开始回测`
8. 弹出参数框后，重点确认：
   - `signal_window`
   - `ema_distance_factor`
   - `risk_reward_ratio`
   - 其他策略参数
9. 回测完成后，看：
   - 统计表
   - 成交记录
   - K线图表

### 回测完成后

- 结果会自动同步到 `Brooks图表 -> 记录`
- 在 `Brooks图表` 里选中对应记录，就能切到对应合约和时间窗
- 后续知识点优化统一顺序：
  1. 在 `知识点` 里选大类
  2. 再选细项
  3. 在图表里看实现结果
4. 逐条讨论和修正

## 5. 当前先核对的知识点：背景 -> 突破

当前程序口径：

1. 底层结构单独分类：
   - `震荡`
   - `窄幅通道`
   - `宽幅通道`
2. `突破` 不再作为底层结构，而是单独的事件层
3. 事件层分成：
   - `突破起爆`
   - `突破跟进`
   - `突破测试`
   - `失败突破`
4. 常态市场循环优先按：
   - `TR -> BO -> Tight CH -> Broad CH`
5. 特殊情况先不抢着扩：
   - `TR 里的失败突破`
   - `宽幅通道里的晚突破`
   - `窄幅通道里的二次起爆`
   - `直接从 BO 失败回 TR`
6. `突破起爆`
   - 最近 3 根 K 线里至少 2 根是同向强趋势棒，或者已经形成 3 根连续趋势棒
   - 当前 K 线必须真正收在突破位外侧
   - 突破前必须先进入 breakout mode / 小平衡
7. `突破跟进`
   - 只认起爆后的前两根同向延续棒
   - 仍要站在突破位外侧，且不能出现强反向 bar
8. `突破测试`
   - 只认起爆后的第一次回测
   - 通常回到突破位或 `EMA20` 一带
   - 收盘不能深回原区间
9. `失败突破`
   - 强反向 trend bar
   - 或收盘重新深回原区间
10. 这里已经去掉没有原文支持的“短冷却”条数规则，改成 Brooks 的 breakout mode / 反转次数语境
