# 数据验证与知识点经验总结

## 目的

- 这份文档不是逐项修正清单。
- 它先回答两个问题：
  - 当前主库里的关键研究数据，哪些已经能当主力基准用。
  - 当前已开发知识点，哪些已经形成稳定骨架，哪些仍然只是第一版程序口径。
- 后续逐项优化时，以这份文档作为基线，不再重复从零判断。

## 当前主库关键数据验证

### 当前最关键的数据基准

- `ES.CME 1m`
  - 来源：`Databento / GLBX.MDP3 / ES.c.0 / ohlcv-1m`
  - 当前主库根数：`5,474,109`
  - 当前主库范围：`2010-06-07 08:00:00` 到 `2026-04-02 16:35:00`
  - 质量结论：`通过`
- `ETHUSDT.GLOBAL 1m`
  - 来源：`Binance`
  - 当前主库根数：`3,284,910`
  - 当前主库范围：`2020-01-01 08:00:00` 到 `2026-03-31 12:29:00`
  - 质量结论：`通过`

### 已从主库移除的数据

- `SPY.SMART 1m`
  - 原因：股票分钟流省略零成交分钟，容易和 `ES` 期货连续会话口径混淆
- `US500.OTC 1m`
  - 原因：`CFD 24h` 连续图和 `ES` 的日内语境并不一致

### 日级根数分布

- `ES.CME`
  - 最小日根数：`57`
  - `P10`: `300`
  - 中位数：`1360`
  - `P90`: `1380`
  - 最大日根数：`1396`
  - 解释：
    - 这是期货连续会话加假期/半日休市混合后的正常表现。
    - 它不是 `1440` 根的 24 小时 CFD 口径。
- `ETHUSDT.GLOBAL`
  - 中位数：`1440`
  - 与加密 `24/7` 语义一致。

### 对数据的实际判断

#### `ES.CME` 现在可以当主力回测数据

- 原因：
  - 现在已经不再只是“计划换 ES”，而是主库里已经有 `ES.CME 1m`。
  - 当前质量检查结果为 `通过`。
  - `异常价`、`异常高低`、`重复时间戳` 都是 `0`。
- 注意：
  - `Databento` 当前对最近一小段 `GLBX.MDP3` 数据有许可边界。
  - 脚本已经自动按许可边界收窄 `end`，所以当前主库不是“实时到此刻”，而是“当前账号许可下可用的最新端点”。
  - `Databento` 的 dataset condition 里还有 `26` 个 `degraded` 日期，属于源数据元信息，不等于整库坏掉。

#### `ETHUSDT.GLOBAL` 继续保留，但主要服务于策略与公共逻辑验证

- 原因：
  - 加密是当前 CTA 与公共逻辑最稳定的现成样本。
  - 但它不适合直接承担美股/股指 `RTH` 语义。
- 结论：
  - 用它验证：
    - 公共逻辑是否跑通
    - CTA 主线是否被打坏
  - 不用它去裁决 `Opening Reversal`、`Midday Reversal` 这类强时段语义知识点。

### 我对当前知识点的分层判断

#### A 类：公共骨架已经合理，可以继续沿用

- 背景层：
  - 背景总览
  - 窄幅通道
  - 宽幅通道
  - 趋势交易区间
  - 震荡
  - 突破
- 关键位置层：
  - EMA20
  - 前高前低 / 波段极值
  - 趋势线 / 通道线
  - 更高周期关键位
  - 开盘价 / 昨收 / 昨高低
- 辅助层：
  - Bar Count
  - ii / ioi / oo
  - 微缺口
  - Open BOM / ORBO / Bar 18

判断理由：

- 这些知识点都已经纳入同一套公共背景层、事件层、EMA、摆点和通道线体系。
- 它们现在更像“统一分析骨架”，而不是彼此割裂的小功能。
- 即使后面还会调参数，也不需要推翻结构。

#### B 类：理论支持是够的，但程序口径还要继续收紧

- `Leg1=Leg2 / AB=CD`
- `Leg1=Leg2：强趋势深回调`
- `Leg1=Leg2：交易区间内部`
- `Leg1=Leg2：与 EMA 配合`
- `交易区间高度 MM`
- `突破高度 / 实体高度 MM`
- `Measuring Gap MM`

判断理由：

- `Leg1=Leg2`
  - 对应原文支撑主要来自：
    - `page-1726`
    - `page-1727`
    - `page-1751`
  - 这些页明确支持：
    - 强趋势深回调时，`Leg1=Leg2` 可作为最小目标
    - 趋势较弱、位于交易区间内部时，`Leg1=Leg2` 常是反弹末端
  - 但当前代码仍然会在部分强趋势尾段留下“几何成立、语境偏弱”的标记。
- `Measuring Gap MM`
  - 对应百科图例支撑主要可见于：
    - `page-0291`
    - `page-0316`
  - 这两页图都体现了同一个核心点：
    - 不是单纯因为出现 gap 就算 `Measuring Gap`
    - 它更依赖 breakout、breakout test、pullback 和 gap 保持不被完全回补的语境
  - 当前程序已经把 `breakout point`、`pullback`、`midline` 和 `Negative MG` 拆开了，这是正确方向
  - 但仍需要继续收紧，尤其要压掉“只满足形状，不满足语境”的情况

#### C 类：目前更像审计层近似，不适合直接入策略

- `Negative Measuring Gap`
- `Measuring Gap middle line 多口径`
- `Opening Reversal`
- `Midday Reversal`

判断理由：

- `Negative Measuring Gap`
  - 现在更像“低可靠性分支提示”，适合作为图表审计辅助
  - 还不适合直接作为 CTA 触发条件
- `Measuring Gap middle line 多口径`
  - 当前是第一版程序近似
  - 它的价值主要在于提醒“这里存在多个可能的中线口径”
  - 不应默认当成稳定交易目标
- `Opening Reversal` 与 `Midday Reversal`
  - 在 `ES` 数据上才真正具备继续精修的基础
  - 在加密或 `CFD 24h` 数据上，只能算语义近似

### 当前知识点体系的总体评价

- 现在的系统已经不是“堆标签”，而是形成了一个统一公共骨架。
- 这点是成立的，值得保留。
- 但它还没有到“所有知识点都能直接进策略”的阶段。
- 更准确地说：
  - 背景、关键位置、辅助层已经可以作为长期底座
  - `Leg1=Leg2` 与 `Measuring Gap` 已经进入“可审计、可收紧”的阶段
  - `Opening/Midday Reversal` 仍在“先用 ES 真语境收口”的阶段

## 当前经验结论

### 数据侧

- `ES.CME 1m` 已经可以作为主力回测与知识点对照基准。
- `SPY.SMART` 与 `US500.OTC` 已从主库移除，避免图表和资料比对时混口径。
- `ETHUSDT.GLOBAL` 继续用于程序主线与 CTA 主线稳定性验证。

### 知识点侧

- 先不要继续扩新知识点。
- 当前最值得继续优化的只有两条主线：
  - `Leg1=Leg2`
  - `Measuring Gap MM`
- 优化顺序应是：
  - 先在 `ES.CME` 上逐图核
  - 再压误报
  - 再决定哪些进入 CTA

### 策略侧

- 当前已进 CTA 的仍然只是 3 套策略主线。
- 这不是问题，反而是好事。
- 因为如果 `Leg1=Leg2` 和 `Measuring Gap MM` 还没收干净，就不应该急着推进策略触发。

## 下一步建议

1. 以 `ES.CME 1m` 为主图，重新做 `Leg1=Leg2` 的逐图误报校验。
2. 用百科图页继续核 `Measuring Gap MM`、`Negative MG`、`middle line` 的显示口径。
3. 把 `A 类 / B 类 / C 类` 作为后续知识点优化优先级，不再并行扩新项。
