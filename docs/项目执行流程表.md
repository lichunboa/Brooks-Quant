# 项目执行流程表

## 用途

这个表是给“新窗口接手”用的。

如果后面你重新开一个窗口，只要先看这份表，就能快速知道：

- 当前项目主线在哪一层
- 哪些模块已经落地
- 下一步应该改哪里
- 哪些东西是生成物，不要误当源码改

## 总流程

| 层级 | 名称 | 主要文件/入口 | 当前状态 | 下一步通常做什么 |
| ---- | ---- | ------------- | -------- | ---------------- |
| 1 | 资料层 | `策略资料/` | 已建立 | 查理论原文、定位课程章节、找百科/Ali 补充 |
| 2 | 公用知识体系层 | `brooks_chart_app/catalog.py` | 已建立 | 补课程条目、补百科/Ali 公共主题、补图表映射 |

## 回测纪律

- 所有策略回测必须走 `vnpy_ctastrategy` 的 `BacktestingEngine` 或界面里的 `CTA回测`。
- 禁止自写独立撮合逻辑、自写回测成交引擎、自写收益曲线引擎来替代 `CTA回测`。
- `backtests/` 里的脚本只允许做：
  - 参数组织
  - 批量调度
  - 结果导出
  - 结果同步回 `Brooks图表`
- 不允许把脚本本身做成另一套回测系统。

## 当前已接入 CTA 的策略

- `H1/L1 首次回调顺势`
- `H2/L2 二次入场顺势`

## 当前单一主干

当前建议始终沿这条主干开发：

`资料 -> 公用知识体系 -> 图表映射 -> 策略蓝图 -> CTA策略 -> 回测/复盘`

## 当前唯一仍存在的显式分叉

当前还有一处需要后续继续收口：

- 图表 setup 生成
- CTA 策略 setup 生成

但这两条现在都已经共用：

- `brooks_chart_app/setup_engine.py`
- `analyze_brooks_context()`
- `is_signal_context_supported()`

所以它们已经不是看不见的旧分支，而是“已显式统一到一个共享模块上的双入口”。

## 当前固定 14 步模板

每个策略蓝图都按这 14 步组织：

1. 背景
2. 关键位置
3. setup 前提
4. signal bar 类型
5. entry trigger
6. 触发失效
7. 初始止损类型
8. 实际风险
9. 仓位与杠杆
10. 第一目标
11. partial / scalp / swing
12. BE 条件
13. 提前离场
14. re-entry / add-on
