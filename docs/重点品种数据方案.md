# 重点品种数据方案

## 当前结论

- `US500.OTC` 继续保留，用于程序开发、界面调试和非时段敏感的初步核对。
- `ES.CME` 定为后续 Brooks 对照、截图复盘、策略优化的优先基准数据。
- 真正“免费 + 高质量 + 1 分钟级 + 可长期稳定使用”的 `ES` 历史数据，当前不现实。
- 当前建议的实际接入顺序：
  - `IBKR + CME 订阅`：适合日常研究和后续半自动补数。
  - `Databento`：适合补长历史、批量导出、统一 CSV 落库。
  - `CME DataMine`：适合补非常久的正式历史，但成本最高。
- 详细选型说明见：
  [`/Users/mitchellcb/Desktop/量化交易vnpy/docs/ES数据接入选型.md`](/Users/mitchellcb/Desktop/量化交易vnpy/docs/ES数据接入选型.md)

## 当前统一口径

- 加密现货分钟线：`Binance` 公共 K 线接口
- 外汇、贵金属、指数 CFD 分钟线：`Dukascopy`
- 仓库内部符号统一使用：
  - 加密：`BTCUSDT`、`ETHUSDT`、`BNBUSDT`、`SOLUSDT`、`XRPUSDT`、`ADAUSDT`
  - 外汇与金属：`EURUSD`、`GBPUSD`、`USDJPY`、`AUDUSD`、`USDCHF`、`USDCAD`、`XAUUSD`、`XAGUSD`
  - 指数：`US500`、`NAS100`

补充：

- `ES` 不走免费公共接口。
- `ES` 统一使用仓库内部符号：`ES`
- `ES` 统一使用交易所：`CME`
- `ES` 优先导入连续研究数据或主力连续导出，不建议把近月、次近月混在同一个连续符号里。

## 推荐用法

正式库未被占用时，直接写入主库：

```bash
cd /Users/mitchellcb/Desktop/量化交易vnpy
uv run python scripts/import_dukascopy_to_duckdb.py --symbols otc --start 2020-01-01 --end 2026-04-01 --interval 1m
uv run python scripts/import_binance_spot_to_duckdb.py --symbols crypto --start 2020-01-01 --end 2026-04-01 --interval 1m
uv run python scripts/check_bar_data_quality.py --symbols all --interval 1m --database-file database.duckdb
```

如果 vn.py 图形界面正在运行，主库会被 DuckDB 锁住。这时先写到暂存库：

```bash
cd /Users/mitchellcb/Desktop/量化交易vnpy
uv run python scripts/import_dukascopy_to_duckdb.py --symbols otc --start 2020-01-01 --end 2026-04-01 --interval 1m --database-file database_import_staging.duckdb
uv run python scripts/import_binance_spot_to_duckdb.py --symbols crypto --start 2020-01-01 --end 2026-04-01 --interval 1m --database-file database_import_staging.duckdb
uv run python scripts/check_bar_data_quality.py --symbols all --interval 1m --database-file database_import_staging.duckdb
```

导入 `ES` 的推荐方式是先从已选平台导出 `CSV`，再落到 DuckDB：

```bash
cd /Users/mitchellcb/Desktop/量化交易vnpy
uv run python scripts/import_generic_csv_to_duckdb.py \
  --csv /绝对路径/es_1m.csv \
  --symbol ES \
  --exchange CME \
  --interval 1m \
  --datetime-column datetime \
  --open-column open \
  --high-column high \
  --low-column low \
  --close-column close \
  --volume-column volume \
  --timezone UTC \
  --database-file database_import_staging.duckdb

uv run python scripts/check_bar_data_quality.py --symbols ES --interval 1m --database-file database_import_staging.duckdb
```

如果使用 `Databento`，可以先估算成本，再直接下载并写入 DuckDB：

```bash
cd /Users/mitchellcb/Desktop/量化交易vnpy
export DATABENTO_API_KEY=你的Key

uv run python scripts/import_databento_to_duckdb.py \
  --dataset GLBX.MDP3 \
  --symbols ES.c.0 \
  --schema ohlcv-1m \
  --stype-in continuous \
  --start 2018-01-01T00:00:00Z \
  --estimate-only

uv run python scripts/import_databento_to_duckdb.py \
  --dataset GLBX.MDP3 \
  --symbols ES.c.0 \
  --schema ohlcv-1m \
  --stype-in continuous \
  --start 2018-01-01T00:00:00Z \
  --output-symbol ES \
  --database-file database_import_staging.duckdb
```

## 质量检查重点

- 是否存在概览记录
- 开始时间、结束时间、总根数
- 非法价格
- `OHLC` 区间错误
- 重复时间戳
- 可疑时间缺口

## 当前注意事项

- `Dukascopy` 的 `--end` 是开区间，想包含 `2026-03-31`，要写成 `--end 2026-04-01`
- `NAS100` 支持别名 `US TECH 100`
- `US500` 支持别名 `US 500`
- 外汇、金银、指数在周末或停盘段出现空白是正常现象，质检报告会列出缺口数量，但要结合品种交易时段解释
- `ES` 期货要单独结合 `Globex` 和后续是否使用 `RTH` 模板解释缺口，不能拿 `US500 CFD` 的 24 小时口径直接类比
