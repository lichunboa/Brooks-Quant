### 策略开发

- 每个策略是一个单独蓝图。
- 蓝图只做三件事：
  - 说明它属于哪一类策略。
  - 列出它引用哪些公用知识点。
  - 写该策略当前的微调方向和代码路径。

## 六层主流程

当前建议你后续都按这六层看项目：

1. 资料层
2. 公用知识体系层
3. 图表映射与审计层
4. 策略开发蓝图层
5. CTA 策略执行层
6. 回测 / 导出 / 复盘层

主干是单一的：

`资料 -> 公用知识体系 -> 图表映射 -> 策略蓝图 -> CTA策略 -> 回测/复盘`

当前唯一需要继续收口的一处分叉，是“图表信号生成”和“CTA 策略信号生成”仍是双实现，但两者已经共享同一个背景分析入口，不再是隐形旧分支。

## 原则

- 公用知识点先整理完整，再谈策略落地。
- 策略蓝图只引用公用知识点，不反向污染公用目录。
- 图表映射属于公共概念的代码落地，不属于某个策略私有目录。
- 任何新策略，都应该先在 `策略开发` 模块里建立蓝图，再决定写不写代码。

## 策略开发固定模板

每个策略蓝图都固定展示以下 14 个环节：

1. 背景
2. 关键位置
3. setup 前提
4. signal bar 类型
5. entry trigger
6. 触发失效
7. 初始止损类型
8. 实际风险
9. 仓位与杠杆
10. 第一目标
11. partial / scalp / swing
12. BE 条件
13. 提前离场
14. re-entry / add-on

这样做的目的不是把公用知识点重新做成模板，而是让每个策略去引用公用知识点后，再按同一套开发框架做微调。

## 开发顺序建议

1. 继续补完公用知识体系的课程条目和百科补充。
2. 补策略开发页里的蓝图说明和引用关系。
3. 再选择一个策略做编码。
4. 编码后，把对应公共概念接回图表审计层。
